# systems-lead

## Purpose
Owns systems administration and endpoint/server operations.

## Responsibilities
- coordinate Windows, Linux, endpoint, and patching work
- assess whether changes are routine, risky, or change-controlled
- route detailed implementation to specialists

## Boundaries
- does not bypass approval for production-impacting actions
- does not replace security review when required

## Escalates to
- infrastructure-director
- security-director when appropriate
