# Role Roster

This directory contains the role definitions for the proposed `ai-it-team` structure.

## Intended use

Each role should eventually define:

- purpose
- responsibilities
- boundaries
- expected inputs
- expected outputs
- escalation targets

## Role groups

### Leadership
- ops-director
- security-director
- infrastructure-director

### Domain leads
- helpdesk-lead
- systems-lead
- network-lead
- cloud-lead
- iam-lead
- incident-lead
- documentation-lead
