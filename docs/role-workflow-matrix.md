# Role-to-Workflow Matrix

## Purpose

This matrix shows how the role roster maps onto the core workflows.

It is not a strict RACI table yet, but it provides a practical starting point for understanding who does what.

## Support triage

Primary:
- helpdesk-lead
- endpoint-tech
- systems-lead

Secondary:
- network-lead
- iam-lead
- documentation-lead

Escalation:
- ops-director

## Access / IAM changes

Primary:
- iam-lead
- iam-specialist
- onboarding-offboarding-specialist

Secondary:
- security-director
- documentation-lead

Escalation:
- human operator

## Incident response

Primary:
- incident-lead
- incident-triage-specialist

Secondary:
- systems-lead
- network-lead
- cloud-lead
- monitoring-specialist
- documentation-lead

Escalation:
- ops-director
- human operator

## Infrastructure changes

Primary:
- systems-lead
- cloud-lead
- relevant specialists

Secondary:
- infrastructure-director
- security-director
- documentation-lead

Escalation:
- human operator

## Documentation maintenance

Primary:
- documentation-lead
- runbook-writer
- change-log-specialist

Secondary:
- relevant domain leads

Escalation:
- none by default unless a process conflict is discovered

## Why this matters

The project should make role participation visible so that workflows feel coordinated rather than implied.
