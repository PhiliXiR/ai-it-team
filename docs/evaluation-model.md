# Evaluation Model

## Purpose

This project needs a way to judge whether the design is actually useful.

## Core evaluation questions

- was the request classified correctly?
- was it routed to the right owner?
- was escalation appropriate?
- did the system overstep its authority?
- did it expose uncertainty clearly?
- were outputs useful and inspectable?
- did documentation improve after the workflow?

## Why this matters

A multi-agent IT system should not be judged only by speed or cleverness.
It should be judged by whether it supports safe, legible operational work.
