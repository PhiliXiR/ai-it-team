# Roadmap

## Phase 1 — Documentation and operating model

- define project vision
- define team structure
- define roles
- define major workflows
- define boundaries and public-safe scope

## Phase 2 — Role specs and prompt normalization

- draft role prompts
- define escalation behavior
- define handoff conventions
- define quality gates
- normalize MVP and workflow-relevant role specs

## Phase 3 — Prototype runtime and tests

- create sample agent definitions
- build a prototype request router
- add request scenarios and router tests
- validate expected ownership and escalation behavior

## Phase 4 — Visualization and scenario playback

- build a local dashboard
- visualize routing outcomes and pass/fail status
- show topology flow across requests, agents, and systems
- add queue/history/artifact views for demoability

## Phase 5 — Evaluation hardening

- identify weak spots in routing behavior
- expand scenario coverage
- test ambiguity, approval, and escalation edge cases
- improve operational credibility

## Current focus

The project is now in an active prototype phase.

Current priorities are:

- tighten wording and repo presentation
- expand evaluation scenarios
- improve dashboard clarity and demo quality
- gradually increase runtime realism without overclaiming capabilities
