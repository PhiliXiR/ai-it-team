# Artifacts and Outputs

## Purpose

A useful team produces more than answers. It produces reusable operational artifacts.

## Core output types

- triage summary
- routing recommendation
- access review note
- change plan
- rollback plan
- incident timeline
- incident summary
- runbook draft
- change log entry
- postmortem draft

## Why this matters

These outputs make the system:

- inspectable
- easier to review
- easier to document
- more operationally useful

The system should be judged partly by the quality of these artifacts.
