# Support Triage Workflow

## Goal

Provide a repeatable path for handling incoming support issues.

## Trigger

A user reports an issue, request, or symptom.

## Flow

1. **helpdesk-lead** classifies the request
2. missing context is gathered
3. request is routed to the most likely owning domain
4. specialist or lead investigates
5. result is documented

## Required outputs

- issue classification
- owner assignment
- collected context
- resolution or escalation path

## Common destinations

- systems-lead
- network-lead
- iam-lead
- incident-lead

## Human checkpoints

- if the request implies risky production action
- if approval is required
- if identity or access ambiguity changes risk
