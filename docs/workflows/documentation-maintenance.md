# Documentation Maintenance Workflow

## Goal

Turn repeated operational work into usable documentation.

## Trigger

A workflow, decision, or recurring operational pattern is missing, unclear, or stale in documentation.

## Flow

1. **documentation-lead** identifies missing or stale material
2. **runbook-writer** drafts structured documentation
3. relevant technical owner validates details
4. **change-log-specialist** records significant updates when needed

## Required outputs

- runbook or workflow draft
- assumptions and unknowns
- owner validation state
- update record if applicable

## Human checkpoints

- when documentation implies policy decisions
- when uncertainty remains high
- when the workflow affects sensitive production operations
