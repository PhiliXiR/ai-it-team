# Change Classes

## Purpose

Not all changes should be treated the same.

## Suggested classes

### Standard change
Routine, low-risk, repeatable work.

### Normal change
A non-trivial change requiring planning and review.

### High-risk change
A change with meaningful production, security, or availability risk.

### Emergency change
A change made under incident pressure or urgent operational need.

## Why this matters

Classifying changes makes it easier to tie them to:

- approval expectations
- documentation expectations
- rollback expectations
- who needs to review them
