# Example: Incident Coordination Flow

## Scenario

Multiple users report that an internal line-of-business system is unreachable.

## Example flow

1. **incident-triage-specialist** gathers first symptoms
2. **incident-lead** assigns likely severity and scope
3. likely owners are identified:
   - systems
   - network
   - cloud
4. **monitoring-specialist** helps interpret available alert data
5. **incident-lead** keeps the response coordinated
6. **documentation-lead** or **change-log-specialist** captures the timeline

## Why this example matters

It shows that the team model is not just for static documentation. It is meant to coordinate cross-domain response work.
