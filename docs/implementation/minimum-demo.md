# Minimum Demo

## Goal

The first credible demo should prove the system can:

- classify a request
- route it to the right role
- escalate when appropriate
- produce a useful artifact

## Suggested first demo scenario

A support request involving VPN access.

Why this is good:

- easy to understand
- touches multiple possible domains
- demonstrates routing and escalation
- does not require a giant infrastructure simulation

## Minimum demo outputs

- request classification
- role handoff chain
- escalation point if applicable
- resolution note or runbook update
