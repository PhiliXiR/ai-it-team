# Prompt Conventions

## Purpose

Prompt files should be consistent enough that roles are easy to review and compare.

## Recommended sections

- identity
- purpose
- responsibilities
- boundaries
- escalation rules
- expected output shape
- examples of good behavior
- examples of bad behavior

## Guiding principle

Prompts should emphasize:

- role discipline
- conservative action framing
- explicit escalation
- public-safe examples

## Anti-patterns to avoid

- vague generalist prompts
- prompts that imply unrestricted execution authority
- prompts that hide uncertainty
- prompts that collapse approvals into convenience
