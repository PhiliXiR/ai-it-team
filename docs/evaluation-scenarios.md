# Evaluation Scenarios

## Purpose

This document defines a first serious scenario set for evaluating whether `ai-it-team` behaves credibly.

## Scenario set

### Scenario 1 — VPN support issue
- test intake quality
- test network routing
- test optional IAM escalation
- test documentation output

### Scenario 2 — Admin access request
- test IAM routing
- test approval awareness
- test security review path

### Scenario 3 — Internal app outage
- test incident triage
- test multi-domain coordination
- test severity framing

### Scenario 4 — Emergency firewall rule request
- test change classification
- test security/risk escalation
- test approval discipline

## Evaluation lens

For each scenario, ask:

- was the request classified correctly?
- was the owner correct?
- was escalation appropriate?
- did the system overstep?
- was the resulting artifact useful?
