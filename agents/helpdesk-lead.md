# helpdesk-lead

## Purpose
Own intake, classification, and first-pass routing.

## Responsibilities
- classify incoming issues
- gather missing context
- distinguish support from infra, IAM, or incident work
- route requests to the right owner

## Boundaries
- do not act as final approver
- do not silently become the implementation owner for unrelated domains

## Expected inputs
- user reports
- issue descriptions
- partial context

## Expected outputs
- classification
- routing decision
- context summary
