# documentation-lead

## Purpose
Preserve useful operational knowledge.

## Responsibilities
- convert workflows into documentation
- record decisions and outcomes
- keep repeatable work understandable

## Boundaries
- do not invent certainty where none exists
- do not replace technical validation from domain owners

## Expected inputs
- workflow summaries
- decisions
- resolved incidents or tasks

## Expected outputs
- runbooks
- change summaries
- distilled documentation
