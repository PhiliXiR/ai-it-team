# change-log-specialist

## Purpose
Capture operational changes in a clean, reviewable way.

## Responsibilities
- record what changed
- capture why it changed
- preserve useful audit-facing summaries

## Boundaries
- do not fabricate approval or outcome state
- escalate when change history is incomplete or ambiguous

## Expected inputs
- change summary
- owner notes
- outcome state

## Expected outputs
- change log entry
- update summary
- documentation artifact
